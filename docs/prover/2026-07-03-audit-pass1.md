F1 — A quick NEWEST version hides a full analysis that still exists on the track's page.
> "the row is the newest version STRICTLY … every catalog cell … is read from the newest version's most-complete run, never blended across versions" — SPEC §D, D-INV-35
Consequence: a producer whose newest bounce was a quick run, but whose previous version had a full analysis
(a real lean), sees the catalog reference cell read "full analysis only" (i.e. no lean) for that track — even
though a full analysis with a lean is one click away in the track's plaque. WHO: the producer scanning the
library; WHAT THEY SEE: an empty/"full analysis only" reference cell that reads as "no lean available" when a
lean actually exists on an older version. The strict-newest decision is recorded, but its user-facing
consequence is not.
Action: add a spec sentence + (when the reference column ships) a cell hint: when the newest version is
quick-only BUT an older version of the same track has a full-run lean, the cell reads "full analysis on an
earlier version →" (links to the plaque), not a bare "full analysis only". Recommend this over silently
dropping the lean.
`should-clarify · confusing-for-users (cognitive-load)` — **status: NOT folded — spec sentence pending; low
urgency (reference column itself is deferred/not-built, so no live user impact yet). Queue behind the column ship.**
